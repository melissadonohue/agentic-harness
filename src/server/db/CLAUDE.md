# src/server/db/ — Database Conventions (Drizzle + Postgres)

This directory is the database vendor seam. It isolates all database access, schema definitions, and migrations behind a stable interface. The default is Drizzle ORM with Postgres.

## Purpose

- Define the database schema using Drizzle's TypeScript-first schema definition.
- Generate and manage migrations via `drizzle-kit`.
- Provide a typed repo layer for all data access.
- Export the database client for use across the application.

## Directory Structure

```
src/server/db/
  index.ts         — Database client export and connection setup
  schema/
    index.ts       — Re-exports all entity schemas
    <entity>.ts    — One file per entity (e.g., users.ts, projects.ts)
  repos/
    <entity>.ts    — Typed CRUD functions per entity
  migrations/      — Generated by drizzle-kit. Never hand-edit.
  CLAUDE.md        — This file
```

## How to Add an Entity

This is the full end-to-end process for adding a new database entity:

### 1. Create the Schema File

Create `schema/<entity-name>.ts` with the Drizzle table definition:

```typescript
// schema/projects.ts
import { pgTable, text, timestamp, uuid } from 'drizzle-orm/pg-core';

export const projects = pgTable('projects', {
  id: uuid('id').primaryKey().defaultRandom(),
  name: text('name').notNull(),
  description: text('description'),
  ownerId: uuid('owner_id').notNull(),
  createdAt: timestamp('created_at', { withTimezone: true }).notNull().defaultNow(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).notNull().defaultNow(),
});
```

### 2. Export from Schema Index

Add the export to `schema/index.ts`:

```typescript
export { projects } from './projects';
```

### 3. Generate the Migration

```bash
npx drizzle-kit generate
```

This creates a migration file in `migrations/`. Review the generated SQL but do not hand-edit it. If the migration is wrong, fix the schema and regenerate.

### 4. Create the Repo Layer

Create `repos/<entity-name>.ts` with typed CRUD functions:

```typescript
// repos/projects.ts
import { eq } from 'drizzle-orm';

import { getDb } from '@/server/db';
import { projects } from '@/server/db/schema';

export async function getProjectById(id: string) {
  const db = getDb();
  const [project] = await db.select().from(projects).where(eq(projects.id, id));
  return project ?? null;
}

export async function createProject(data: { name: string; description?: string; ownerId: string }) {
  const db = getDb();
  const [project] = await db.insert(projects).values(data).returning();
  return project;
}
```

### 5. Create the API Route Handler

Create `src/app/api/<entity-name>/route.ts` with Zod validation:

```typescript
import { NextRequest, NextResponse } from 'next/server';
import { z } from 'zod';

import { requireSession } from '@/server/auth';
import { createProject, listProjects } from '@/server/db/repos/projects';

const createProjectSchema = z.object({
  name: z.string().min(1).max(255),
  description: z.string().optional(),
});

export async function GET() {
  const session = await requireSession();
  const projects = await listProjects(session.userId);
  return NextResponse.json(projects);
}

export async function POST(request: NextRequest) {
  const session = await requireSession();
  const body = await request.json();
  const parsed = createProjectSchema.safeParse(body);

  if (!parsed.success) {
    return NextResponse.json({ error: parsed.error.flatten() }, { status: 400 });
  }

  const project = await createProject({ ...parsed.data, ownerId: session.userId });
  return NextResponse.json(project, { status: 201 });
}
```

### 6. Add Tests

- Unit test for repo functions in `tests/unit/`.
- Integration test with real database in `tests/integration/`.

## Schema Conventions

- **Table names**: `snake_case`, plural (e.g., `projects`, `team_members`).
- **Column names**: `snake_case` (e.g., `created_at`, `owner_id`).
- **Every table** has these columns:
  - `id` — UUID primary key with `defaultRandom()`.
  - `createdAt` — Timestamp with timezone, `defaultNow()`.
  - `updatedAt` — Timestamp with timezone, `defaultNow()`.
- **Foreign keys**: Use `uuid` type matching the referenced table's `id`. Name the column `<referenced_entity_singular>_id` (e.g., `owner_id`, `project_id`).
- **Enums**: Use Drizzle's `pgEnum` for columns with a fixed set of values.

## Migrations

- Generated by `npx drizzle-kit generate`. Stored in `migrations/`.
- **Never hand-edit** generated migration files. If a migration is wrong, fix the schema and regenerate.
- To apply migrations: `npx drizzle-kit migrate` (or via the seed script in dispatch environments).
- Migration files are committed to the repository and applied in order.

## Containment Rules

- No raw SQL outside this module except in migration files.
- No `drizzle-orm` or `drizzle-kit` imports outside `src/server/db/` and `drizzle.config.ts`.
- The rest of the application uses the repo layer functions, never Drizzle query builders directly.
- **Linter gate**: `drizzle-orm` and `drizzle-kit` imports outside allowed paths are build-breaking errors.

## Files

- `index.ts` — Database client. Stub until wired; then exports the Drizzle instance.
- `schema/` — Drizzle table definitions. One file per entity, re-exported from `schema/index.ts`.
- `repos/` — Typed CRUD functions. One file per entity.
- `migrations/` — Generated migration files. Do not hand-edit.
